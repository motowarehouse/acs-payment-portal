# ACS Payment Portal — Update, 13 July 2026

Changes agreed and implemented in this session.

## 1. Copy tracking number
A copy icon next to every tracking number on **Shipments** and **Outstanding** (and the new Outstanding table). Click → copies to clipboard, shows a green check. Tooltip in EN/GR.

## 2. Cheque lifecycle
- New field `Payment.chequeStatus`: **PENDING → CLEARED / BOUNCED** (cheques only). New cheques start PENDING; cheques saved before this update are treated as PENDING.
- **Bounced cheques no longer count toward the COD** — the shipment automatically returns to Awaiting / Partially paid (manual status overrides are respected). This is enforced by a shared `paidSum()` helper used by every page, API and import that sums payments.
- Manage from the **shipment detail page** (pill + ✓ cleared / ✗ bounced buttons on each cheque line) or the **Cheques page** (status column on recent cheques, plus a "pending" counter).
- **Duplicate cheque guard**: saving a cheque whose number (+ bank) already exists returns a warning showing the existing entry; "Save anyway" confirms.

## 3. Outstanding — aging, search, export
- New **Days** column (days since pickup): grey < 14, amber 14–30, red > 30.
- Default sort is now **oldest first** (was largest amount first).
- Search box (tracking number or recipient).
- **Export Excel** button → `/api/export/outstanding` → `Outstanding_COD_YYYY-MM-DD.xlsx`.

## 4. Audit trail & backup
- New `AuditLog` table. Recorded: manual status changes, notes edits, payment additions, payment deletions, cheque status changes, import undos — each with timestamp + login name.
- Shown in a **History** panel on the shipment detail page (last 20 entries).
- **Download backup** button on the Import page → `/api/export/backup` → full Excel workbook (Shipments / Payments / Imports sheets).

## Deployment
`railway.json` already runs `prisma db push` on start, so the new schema (ChequeStatus enum, `chequeStatus` column, `AuditLog` table — all additive, no data loss) applies automatically on the next deploy. Push with `push-to-github.bat` as usual.
